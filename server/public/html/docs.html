Animes Disponibles

La API ofrece acceso a una lista de animes disponibles con detalles como nombre, descripción, episodios y más. Cada anime está identificado por un ID único que permite realizar acciones como ver sus personajes, actualizar la información o eliminarlo de la base de datos.

Personajes de Animes

Los personajes están agrupados dentro de cada anime. Para obtener la lista de personajes de un anime específico, es necesario utilizar el ID del anime en la siguiente ruta:

/api/animes/:animeId/characters

Cada personaje también tiene un ID único, lo que permite acceder a sus detalles específicos, como nombre, biografía, habilidades y más.

Acerca de AnimeDB

AnimeDB es una API pública que proporciona acceso a una base de datos de animes y sus personajes. Está diseñada para ser fácil de integrar en proyectos relacionados con anime y proporciona endpoints sencillos para obtener información sobre animes y sus personajes.

¿Cómo Funciona la API?

La API sigue una arquitectura RESTful. Las solicitudes se realizan utilizando los métodos HTTP estándar (GET, POST, PATCH, DELETE). Los recursos principales de la API son los animes y los personajes. A continuación se detallan los endpoints disponibles:

Endpoints Principales:

  • GET /api/animes: Obtiene una lista de todos los animes disponibles.
  • GET /api/animes/:animeId: Obtiene los detalles de un anime específico usando su ID único.
  • POST /api/animes: Crea un nuevo anime en la base de datos.
  • PATCH /api/animes/:animeId: Actualiza la información de un anime existente.
  • DELETE /api/animes/:animeId: Elimina un anime de la base de datos.

Personajes:

  • GET /api/animes/:animeId/characters: Obtiene la lista de personajes de un anime específico.
  • GET /api/animes/:animeId/characters/:characterId: Obtiene los detalles de un personaje específico.
  • POST /api/animes/:animeId/characters: Crea un nuevo personaje dentro de un anime.
  • PATCH /api/animes/:animeId/characters/:characterId: Actualiza la información de un personaje específico.
  • DELETE /api/animes/:animeId/characters/:characterId: Elimina un personaje específico de un anime.

Ejemplo de Respuesta

Cuando se hace una solicitud a /api/animes/:animeId/characters, la API devuelve una lista de personajes en formato JSON:

{
  "characters": [
    {
      "id": 1,
      "name": "Naruto Uzumaki",
      "role": "Protagonista",
      "abilities": ["Rasengan", "Kage Bunshin no Jutsu"]
    },
    {
      "id": 2,
      "name": "Sasuke Uchiha",
      "role": "Rival",
      "abilities": ["Chidori", "Sharingan"]
    }
  ]
}

Formato de Fechas

Las fechas de estreno de animes y personajes se devuelven en formato ISO 8601, por ejemplo: 2024-01-01T00:00:00Z

Errores Comunes

Si se realiza una solicitud incorrecta o si el anime o personaje no existe, la API devolverá un mensaje de error con el código adecuado:

  • 404 Not Found: El recurso solicitado no existe.
  • 400 Bad Request: La solicitud tiene un formato incorrecto.
  • 500 Internal Server Error: Un error inesperado ha ocurrido en el servidor.
